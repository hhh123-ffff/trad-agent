# 分支保护

在 CI workflow 至少成功运行一次后，用这份清单保护 `main` 分支。

## 推荐规则

创建分支规则或 ruleset，目标分支为：

```text
main
```

启用以下限制：

- 合并前必须创建 Pull Request
- 合并前必须通过状态检查
- 合并前分支必须与目标分支保持最新
- 禁止强制推送
- 禁止删除分支

## 必需状态检查

选择 `.github/workflows/ci.yml` 创建的 GitHub Actions 检查：

```text
后端与前端检查
```

根据 GitHub 页面展示方式，它也可能显示为：

```text
CI / 后端与前端检查
```

## 合并纪律

合并产品改动前，确认 PR 已完成：

- 后端测试通过；
- Python 编译检查通过；
- 前端生产构建通过；
- 简要说明数据源影响和合规边界影响。

本产品必须保持边界：观澜是复盘和信息整理工具，不能变成交易执行、买卖推荐、目标价、仓位建议或收益承诺产品。
