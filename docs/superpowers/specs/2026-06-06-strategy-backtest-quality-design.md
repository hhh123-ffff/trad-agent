# 主板量价启动策略质量评估与回测设计

## 目标

建立可重复运行、可解释、明确展示数据偏差的日线级策略质量评估闭环，用于回答：

- 当前“主板量价启动”规则历史上筛出了多少候选；
- 候选在信号后的 1、3、5、10 个交易日表现如何；
- 哪些规则、阶段和评分区间更稳定；
- 样本量、数据缺口和未来函数风险是否允许形成结论；
- 每日真实扫描信号后续表现是否与历史回放一致。

本阶段只做研究评估和规则诊断，不输出买卖建议、目标价、仓位建议、收益承诺或自动交易指令。

## 已确认决策

- 采用多维评价，不以单一“命中率”作为主要结论。
- 信号在收盘后生成，回测起点使用下一交易日开盘价，避免使用信号日无法成交的价格。
- 采用“历史逐日重放 + 真实信号持续跟踪”双轨模式。
- 第一版只使用日线数据；分钟线、分时均价线和尾盘回踩条件标记为未验证。
- 历史流通市值通过信号日成交额和换手率估算，并明确标记为估算值。
- 同期基准使用可评估主板样本的等权收益，不依赖额外指数历史接口。

## 当前数据基础

当前本地数据库约有：

- 18.7 万条前复权日线；
- 759 只股票；
- 日线日期范围约为 2025-04-28 至 2026-06-01；
- 9 条已保存扫描结果，覆盖 4 个交易日；
- 6 条观察池日志。

已保存真实信号不足以单独评价策略，因此历史逐日重放是第一版主样本来源，真实信号持续跟踪作为长期验证集。

## 方法与偏差边界

### 无未来函数原则

重放某个信号日时，仅允许使用该日及之前的数据计算候选：

- 至少使用截至信号日的 120 根日线；
- 所有均线、量能、压力位和风险指标只从过去窗口计算；
- 不使用当前题材热度回填历史；
- 不使用未来交易日信息决定当日是否入选；
- 下一交易日开盘价仅用于计算信号后的表现，不参与信号生成。

### 历史流通市值估算

当信号日成交额和换手率均有效时：

```text
估算流通市值 = 信号日成交额 / (换手率 / 100)
估算流通市值亿元 = 信号日成交额 / 换手率 / 1,000,000
```

估算值仅用于验证 `50亿-200亿` 过滤范围，并在每条回测信号和汇总中标记：

- `float_cap_source = estimated_from_turnover`
- 成交额或换手率缺失时，该日该股票不进入严格信号样本；
- 不把估算值描述为精确流通市值。

### 已知偏差

- 当前股票池和名称可能带来幸存者偏差，退市股票覆盖不完整；
- 历史 ST 状态缺失，只能排除当前明确为 ST 的股票；
- 历史题材归属和当日活跃题材不可可靠回放；
- 分钟线缺失，无法验证全天均价线上方、尾盘创新高回踩不破；
- 前复权日线适合结构评估，但与真实成交价格存在差异；
- 可用历史约一年，无法代表完整市场周期。

所有回测报告必须展示这些限制。样本不足或数据质量不达标时，结论状态为“置信度低”，不能静默省略。

## 回放样本与信号分组

### 股票池

历史回放从本地 `daily_bars` 中构建股票池：

- 只保留沪深主板代码；
- 每个信号日至少存在 120 根过去日线；
- 信号日成交额、成交量、换手率和 OHLC 必须有效；
- 当前明确为 ST 的股票排除；
- 上市天数按本地首根日线到信号日的跨度计算，不使用当前上市天数回填历史。

### 评价区间

默认回放范围：

- 开始日期：满足 120 根历史日线后的首个可评估交易日；
- 结束日期：数据库最新交易日前至少 10 个交易日，保证 10 日评价成熟；
- 用户可指定日期范围；
- 对尚未成熟的 3、5、10 日表现，按对应 horizon 标记为 `immature`，不纳入该 horizon 汇总。

### 信号分组

严格信号使用现有确定性筛选与评分逻辑，分组展示：

- `潜伏观察`
- `启动确认`

`过热排除` 和 `数据不足` 只进入诊断漏斗，不进入主要收益统计。

历史题材信息不可用时：

- 不使用当前题材归属回填历史；
- 题材得分按历史缺失处理；
- 报告明确说明潜伏观察阶段可能存在漏样本；
- 启动确认阶段仍按结构分和启动分评价。

### 重复信号

同一股票同一阶段可能连续多日重复出现：

- 原始信号全部保存，用于检查持续性；
- 主要统计默认使用去重样本；
- 去重规则：同一股票、同一阶段在 3 个交易日内连续重复，仅保留首次信号；
- 阶段升级后重新计为新信号；
- API 允许切换查看原始样本和去重样本。

## 表现指标

### 单条信号

回测起点：

- `entry_date`：信号后的下一交易日；
- `entry_price`：`entry_date` 开盘价；
- `signal_close`：仅用于诊断，不作为主要回测起点。

对 1、3、5、10 个交易日分别计算：

- `close_return_pct`：第 N 个交易日收盘价相对 entry open 的收益率；
- `max_favorable_excursion_pct`：前 N 日最高价相对 entry open 的最大上涨；
- `max_adverse_excursion_pct`：前 N 日最低价相对 entry open 的最大回撤；
- `benchmark_return_pct`：同期可评估主板样本等权收益；
- `excess_return_pct`：信号收益减去同期等权基准；
- `positive`：收盘收益是否大于 0；
- `outperformed`：是否跑赢同期等权基准；
- `maturity`：`mature | immature | invalid`。

不定义统一“成功交易”或承诺式命中标准。

### 汇总指标

按整体、阶段、总分区间、启动分区间和信号月份汇总：

- 样本数和成熟样本数；
- 平均数、中位数、25/75 分位数；
- 正收益比例；
- 跑赢基准比例；
- 平均最大上涨；
- 平均最大回撤；
- 收益/回撤分布；
- 连续重复信号占比；
- 数据有效率和排除原因分布。

汇总置信度：

- `< 30` 个成熟样本：低；
- `30-99` 个成熟样本：中；
- `>= 100` 个成熟样本：较高；
- 置信度只表示样本稳定程度，不代表未来表现。

- 严格信号；

- 已成熟样本。

该表用于解释“为什么没有足够候选”，避免只展示最终收益。

## 服务与任务流

### 回放引擎

新增独立 `strategy_backtest.py`，职责为：

- 按股票加载完整本地日线；
- 逐日构建仅包含过去数据的窗口；
- 构造历史股票画像和估算流通市值；
- 调用现有 `evaluate_candidate()`，不复制策略规则；
- 记录漏斗、严格信号、重复状态和未来表现；
- 聚合多维统计与限制说明。

回放按股票逐个处理，避免为每个股票日重复查询数据库。第一版本地同步后台线程执行，复用现有扫描任务模式，不引入 Celery 或 Temporal。

### 真实信号持续跟踪

每次 `post_market_replay` 完成后：

- 把当天 `stealth_scan_results` 中的严格候选同步到 `strategy_signal_outcomes`，`origin=live`；
- 刷新历史真实信号的 1、3、5、10 日表现；
- 未成熟 horizon 保留为 `immature`；
- 数据缺失时标记 `invalid`，不伪造收益。

真实信号与历史回放分开汇总，避免把历史模拟结果与真实运行结果混为一谈。

### 错误与恢复

- 单个股票数据异常记录到回放任务错误摘要，任务继续；
- 数据库写入失败使任务失败；
- 回放任务重复运行使用 upsert，不产生重复信号；
- 服务重启时将未完成任务标记失败；
- 用户可重新运行同一日期范围；
- 回放不调用外部行情接口，只使用本地已落库日线，保证可复测。

## API

新增兼容路由：

- `POST /api/strategy/backtests/run`
  - 参数：`start_date?`、`end_date?`、`symbols?`、`repeat_days=3`
  - 返回回放任务。
- `GET /api/strategy/backtests/latest`
  - 返回最近回放任务、汇总、数据质量和限制。
- `GET /api/strategy/backtests/{run_id}`
  - 返回任务详情和汇总。
- `GET /api/strategy/backtests/{run_id}/signals`
  - 参数：`stage?`、`horizon?`、`primary_only=true`、`limit`、`offset`
  - 返回信号及多周期表现。
- `GET /api/strategy/backtests/{run_id}/funnel`
  - 返回过滤漏斗。
- `GET /api/strategy/live-outcomes`
  - 返回真实每日扫描信号的持续跟踪汇总。

不修改现有候选、扫描和日报接口。

## 前端

在“潜伏挖掘”页面增加“策略质量”区域，不新增营销式页面：

- 顶部：最近回测状态、日期范围、样本置信度、运行按钮；
- 主指标：成熟样本数、5 日中位收益、5 日跑赢比例、5 日平均最大回撤；
- 多周期表：1/3/5/10 日平均与中位收益、正收益比例、跑赢比例、最大上涨与最大回撤；
- 阶段对比：潜伏观察与启动确认并排比较；
- 分布视图：收益分布和回撤分布；
- 过滤漏斗：显示每条硬规则淘汰了多少股票日；
- 信号明细：信号日、下一日开盘、评分、阶段、各 horizon 表现、数据限制；
- 真实信号区：与历史回放分开展示；
- 数据限制：固定展示历史题材、历史 ST、分钟线和幸存者偏差说明。

状态与颜色沿用现有交易终端风格：

- 正常/成熟：松绿；
- 运行中：信号蓝；
- 样本不足/数据偏差：琥珀；
- 无效数据/任务失败：风险红。

## 合规

- 页面和 API 使用“表现、样本、跑赢、回撤、候选、观察”等研究词汇；
- 不使用“稳赚、胜率保证、可买、买点、卖点、目标价、建议仓位”等表述；
- 回测报告固定展示：
  - 历史表现不代表未来；
  - 数据存在偏差与缺口；
  - 不构成证券投资建议；
  - 不模拟手续费、滑点和实际成交能力。

## 测试与验收

### 回放计算测试

- 信号日只使用当日及之前数据；
- 下一交易日开盘作为 entry；
- 1/3/5/10 日收益、最大上涨、最大回撤计算正确；
- 成熟与未成熟 horizon 正确区分；
- 同期等权基准和超额收益正确；
- 流通市值估算公式正确；
- 成交额或换手率缺失时样本被排除；
- 连续同阶段 3 日信号仅首日进入主样本；
- 阶段升级后重新进入主样本；
- 重跑不产生重复结果。

### 服务与 API 测试

- 回放任务成功、部分股票失败、整体失败状态正确；
- 未完成任务在服务重启后标记失败；
- 最新汇总、信号分页、阶段过滤、漏斗和真实信号接口正确；
- 数据不足时返回低置信度和限制说明；
- API 文案不包含投资建议。

### 前端测试

- 无回测、运行中、低样本、完成、失败状态均可渲染；
- 1/3/5/10 日切换和阶段对比正确；
- 漏斗、分布和信号明细无横向溢出；
- 桌面和移动端均可查看关键指标和限制说明。

### 手动验收

1. 运行一次历史回放；
2. 确认任务只读取本地日线，不请求外部行情；
3. 检查日期范围、漏斗、样本数和数据限制；
4. 随机抽查一条信号，手工验证下一交易日开盘和 5 日表现；
5. 运行一次盘后闭环，确认真实信号持续跟踪区更新；
6. 确认页面没有买卖、仓位、目标价或收益承诺文案。

## 本阶段完成标准

- 历史逐日重放可重复运行并持久化结果；
- 多维 1/3/5/10 日表现与同期等权基准可查看；
- 过滤漏斗和数据质量限制清晰可见；
- 真实每日扫描信号开始持续跟踪；
- 策略质量区可在 Web 中使用；
- 全量后端测试、前端构建和一次真实回放烟测通过。
